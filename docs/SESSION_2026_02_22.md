## Движок — это пайплайн

```
Сырые данные
  → Классификация на атомы (6 типов)
    → Маппинг полей (тип + подтип + дефолтный display)
      → Примитивы атомов (show/hide/display/color/size/shape/order)
        → Группировка в виджеты
          → Примитивы виджетов (direction/place/size)
            → Формация (layout)
              → Экран
```

Констрейнты — между стадиями, не в конце.
Пресет — виджет с заранее применёнными примитивами. Кастом — дельта поверх пресета, сохраняется как основа в сессии.

## Пресеты — 10 штук

Две группы: продуктовые (разные представления одного entity) и системные (другие entity типы или служебные состояния).

### Продуктовые (core)

| # | Пресет | Что это | Layout | Атомы |
|---|--------|---------|--------|-------|
| 1 | `product_card_grid` | Маленькая карточка в гриде | grid | images(image-cover), name(h3), price(price) — small: 3 атома |
| 2 | `product_card_detail` | Детальная карточка | single | всё что есть, крупно — large: 10 атомов |
| 3 | `product_row` | Строка-список. Фото маленькое слева, название + цена + 2-3 атрибута справа. Компактнее грида, для "покажи всё что есть до 5000" | list | images(thumbnail), name(body), price(price), brand(tag), rating(rating-compact) — direction: horizontal |
| 4 | `product_comparison` | Сравнение 2-4 товаров. Поля выровнены вертикально, различия подсвечены. "Сравни вот эти два" | comparison | name(h3), price(price), rating(rating-compact) + констрейнты выравнивания |

### Как ложится на пайплайн

Продуктовые (1-5) — разные представления одного entity типа. Разница только в наборе атомов, display и layout. Это core движка.

Системные (6-10) — другие entity типы или служебные состояния. Проще, но без них агент выглядит поломанным.

### Примечание по display

`display` сейчас смешивает типографику (h1, h2, body) и компоненты (badge, tag, price-lg). Осознанное решение: оставляем как есть, в будущем можно разделить на два примитива (typography + component) для большей управляемости.

### Примечание по comparison

Comparison — не отдельный шаблон, а пресет карточки + layout "comparison" + констрейнты выравнивания (одинаковые поля, одинаковая высота строк, подсветка лучших значений, max 4 сущности).

## Как тестировать

**Визуальный тестбенч в админке** — страница без LLM:
- Готовые пресеты (виджет + атомы + форматирование)
- Генерация N вариаций комбинаций примитивов
- Прямой вызов visual_assembly с ручными параметрами
- Рендеринг теми же компонентами что в чате

Цель: отсмотреть все вариации, найти где ломается, добавить констрейнты.
